# Azure Cost Analyzer

Azure 비용 데이터를 분석하여 절감 기회를 식별합니다.

## 사용 시점
- 월별/일별 비용 추이 확인
- 리소스 그룹별 비용 분담
- 비용 이상 탐지 (전월 대비 급증)
- 미사용 리소스 식별 (좀비 리소스)

## 워크플로우

### 1단계: 비용 데이터 수집
Azure MCP의 Cost Management 도구 활용:
- 기간 설정 (기본: 최근 30일)
- 그룹화 차원 (구독/리소스 그룹/태그/서비스)
- 통화 (KRW 우선, 없으면 USD)

### 2단계: 분석 및 인사이트
다음 관점에서 분석:

**비용 분포**
- 상위 10개 비싼 리소스
- 서비스 카테고리별 비중 (Compute, Storage, Network, etc.)
- 리전별 분포

**이상 탐지**
- 전월 대비 ±20% 이상 변동
- 갑작스러운 신규 리소스 등장
- 미사용 자원 (디스크 미연결, 사용하지 않는 IP)

**최적화 기회**
- Reserved Instance 후보 (장기 가동 VM)
- 다운사이징 가능 리소스 (낮은 사용률)
- 삭제 가능 리소스 (orphaned, stopped 장기간)
- Spot/Low Priority 전환 후보

### 3단계: 보고서 작성

```
## 💰 Azure 비용 분석 보고서

**기간**: 2026-03-25 ~ 2026-04-24
**총 비용**: ₩X,XXX,XXX (전월 대비 +X.X%)

### 📊 서비스별 비용 Top 5
1. Virtual Machines: ₩XXX,XXX (XX%)
2. Storage: ₩XXX,XXX (XX%)
...

### 🔍 이상 항목
- <리소스명>: 전월 대비 +XX% 증가 (원인 추정: ...)

### 💡 최적화 제안 (예상 절감액)
1. **[높음]** Reserved Instance 적용 → 월 ₩XXX,XXX 절감
2. **[중간]** 미사용 디스크 X개 삭제 → 월 ₩XX,XXX 절감
3. **[낮음]** 개발 환경 야간 셧다운 → 월 ₩XX,XXX 절감

**예상 총 절감액**: ₩XXX,XXX/월
```

## 주의사항
- 비용 데이터는 24-48시간 지연 가능 (Azure Billing 특성)
- 추정치는 명확히 표기 (Reserved Instance 시뮬레이션 등)
- 실제 절감 액션은 cost-analyzer 스킬에서 실행하지 않음 → resource-deploy 스킬로 위임
