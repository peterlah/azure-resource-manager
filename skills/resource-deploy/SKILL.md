# Azure Resource Deploy

Azure 리소스의 생성/수정/삭제(CUD) 작업을 안전하게 수행합니다.

## 사용 시점
- 신규 리소스 생성 (RG, VM, Storage, AKS, AI Foundry 등)
- 기존 리소스 속성 변경 (태그, SKU, 크기 등)
- 리소스 삭제

## ⚠️ 안전 원칙

### 모든 변경 전 사전 확인
1. **현재 상태 보고** — 변경 대상 리소스의 현재 상태를 먼저 조회
2. **변경 사항 명시** — 무엇이 어떻게 바뀌는지 한국어로 요약
3. **명시적 승인 요청** — 사용자가 "진행해" / "yes" 응답 후에만 실행
4. **삭제는 이중 확인** — 삭제 작업은 리소스 이름 재입력 요청

### 절대 금지 사항
- 사용자 확인 없이 변경/삭제 실행
- 프로덕션 리소스 일괄 삭제
- 시크릿/키 평문 출력
- `--yes` 플래그 자동 추가

## 워크플로우

### 생성 작업
1. 사용자 요구사항 파악 (리소스 타입, 이름, 위치, SKU)
2. 네이밍 컨벤션 검증 (governance-check 스킬 자동 호출 권장)
3. 변경 계획 표시 (다음 형식 사용)

```
## 🚀 배포 계획

**작업**: 신규 생성
**리소스**: Storage Account
**이름**: stkoreacentral001
**리소스 그룹**: rg-shared-prod
**리전**: koreacentral
**SKU**: Standard_LRS
**태그**: env=prod, owner=peterlah

진행하시겠습니까? (yes/no)
```

4. 승인 후 Azure MCP로 실행
5. 결과 검증 및 리소스 ID 보고

### 수정 작업
1. 변경 전/후 비교 표 표시
2. Drift 위험 경고 (Terraform 관리 리소스 여부 확인)
3. 승인 후 실행

### 삭제 작업
1. 의존성 검사 (해당 리소스를 참조하는 다른 리소스)
2. 다음 형식으로 경고:

```
## ⚠️ 삭제 확인

다음 리소스를 영구 삭제합니다:
- 이름: <리소스명>
- 타입: <타입>
- 의존 리소스: <개수> (있다면 나열)

복구할 수 없습니다. 삭제하려면 리소스 이름을 정확히 입력하세요.
```

3. 사용자가 정확한 이름 입력 시에만 실행

## Azure CLI 보조 사용
복잡한 작업이나 MCP가 지원하지 않는 작업은 `az` 명령어를 Bash로 실행:
```bash
az resource list --resource-group <rg> --output table
```

단, 이 경우에도 위 안전 원칙은 동일 적용.
