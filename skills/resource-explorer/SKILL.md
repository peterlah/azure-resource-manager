# Azure Resource Explorer

Azure 구독에 있는 리소스를 체계적으로 탐색하고 보고합니다.

## 사용 시점
- 사용자가 Azure 리소스 현황 파악을 원할 때
- 특정 리소스 타입(VM, Storage, AKS 등) 목록이 필요할 때
- 리소스 그룹 구조 파악이 필요할 때
- 태그 기반 리소스 검색이 필요할 때

## 워크플로우

### 1단계: 컨텍스트 확인
먼저 Azure MCP를 통해 현재 활성 구독과 컨텍스트를 확인합니다.
- 다중 구독 환경이면 사용자에게 어느 구독인지 확인
- 인증되지 않았다면 `az login` 안내

### 2단계: 리소스 조회
Azure MCP의 다음 도구를 활용합니다:
- 구독 목록 조회
- 리소스 그룹 목록 조회
- 특정 그룹 내 리소스 조회
- Azure Resource Graph 쿼리 (대규모 검색)

### 3단계: 결과 정리
다음 형식으로 한국어로 보고:

```
## 📊 Azure 리소스 현황

**구독**: <구독명>
**리전 분포**: <리전별 개수>
**리소스 그룹**: <개수>

### 리소스 그룹별 요약
| 리소스 그룹 | 리전 | 리소스 수 | 주요 타입 |
|-----------|------|---------|---------|
| ...      | ...  | ...     | ...     |

### 주의 필요 항목
- <태그 누락, 비표준 네이밍 등>
```

## 한국어 표현 가이드
- "Resource Group" → "리소스 그룹"
- "Subscription" → "구독"
- "Region" → "리전" (지역 X)
- "Tag" → "태그"

## 주의사항
- 대량 조회 시 Azure Resource Graph를 우선 사용 (성능)
- 민감 정보(시크릿, 연결 문자열)는 절대 출력하지 않음
- 읽기 전용 작업만 수행 (생성/삭제는 별도 스킬 사용)
